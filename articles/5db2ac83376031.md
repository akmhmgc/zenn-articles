- 親プロセスで定義した環境変数は子プロセスに引き継がれる（逆はない）

```ruby
fork do
  # 子プロセス内で定義
  ENV['MESSAGE'] = 'wing it!'
  puts "子プロセスから見たMESSAGE: #{ENV['MESSAGE']}"
end

# 親プロセスでは呼ばれない
puts "親プロセスから見たMESSAGE: #{ENV['MESSAGE']}"
```
